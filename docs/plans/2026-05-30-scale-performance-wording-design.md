# Scale & Performance Wording Design

## Goal

Refine the first `Scale & Performance` bullet in `content/cv.md` so it communicates system scale in a more executive tone while preserving concrete numbers.

## Chosen wording direction

- Prefer an executive tone over implementation-heavy phrasing.
- Keep both workload patterns: thousands of events per second for high-throughput pipelines and 25 Hz for video-broadcast streaming workloads.
- Keep the sentence compact and CV-friendly.

## Approved wording

```md
- Systems operate across both high-throughput pipelines processing thousands of events/sec and streaming workloads running at 25 Hz (video broadcast frame rate).
```
